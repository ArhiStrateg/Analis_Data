Данное приложение создано для определения максимально выгодных со стороны прибыльности временных условий для компании.
Основными критериями для поиска являются следующие приложения -
Restaurant, News, Memory, Paint, Wallpapers, AirHockey, PhotoShare, JigsawPuzzle.
Задачей является найти наиболее прибыльные дни недели и наиболее прибыльные интервалы времени получения максимальной прибыли компанией.
Задачa реализованна в двух вариантах расчета.

Вариант 1 (более точный).
Вариант 1 основан следующих этапах:
1 - Создание двух списков дней недели, а именно дни, которые несут максимальную прибыльность, и дни, которые не несут максимальную прибыльность (исключительно для соответсвующего приложения),
1.1 - Подготовка ордеров для расчета - а именно разделение ордеров (переход через значение чч.00 и 24.00),
1.2 - Подготовка значений прибыли разделенных ордеров - исходя из соотношения разделений в п.1.1,
2 - Исходя из полученных двух списков дней недели, создаются два значения среднего значение прибыльности (временной интервал 1 час и только для своего определенного дня недели),
3 - Далее проводится фильтрация - все значения прибыльности (в определенный интервал времени по соответсвующему дню недели для каждого списка из п.1 отдельно) привышающие среднее значение являются прибыльными,
4 - Для отобранных максимальных значений (основываясь на определенном интервале и определенном приложении) - проводим рассчет прибыли,
5 - Основываясь на выводе, что отобранные временные интервалы являются максимально прибыльными, мною добавлен нюанс повышения стоимости услуг в эти моменты (предложено повышение на 1%).

Вариант 2 (менее точный).
Очень схож с итоговым расчетом варианта 1 - но:
1 - Для анализа создается только один список дней недели (исключительно с максимальными показателями - алгоритм отбора идентичен варинту 1),
2 - Отфильтровка промежутков времени несущих максимальные прибыли берется относительно уже всех дней недели (а не созданных ранее списков).
